## 0. Codex 使用规则（必须先读）

本文件是星橡集团官网开发的**唯一产品事实源（Single Source of Truth）**。Codex 或其他开发代理开始工作前，必须先读取根目录 `AGENTS.md`，再完整阅读本 PRD。

### 0.1 冲突优先级

发生冲突时，按以下顺序执行：

1. 本文件 `docs/STAROAK_WEBSITE_MASTER_PRD_V3.4.md`；
2. 根目录 `AGENTS.md`；
3. `docs/PAGE_CONTENT_MATRIX_V3.4.md`、`docs/DESIGN_SYSTEM_V3.4.md`、`docs/CMS_API_DEPLOYMENT_V3.4.md`；
4. 当前代码与 `specs/*.json`；
5. `docs/visual-references/` 中的视觉图片；
6. 更早版本的设计确认稿。

### 0.2 开发原则

- **不要重做品牌、信息架构或主视觉方向。** 在当前基线内优化，不得擅自改变定位、导航、页面职责和核心业务关系。
- **不要根据旧视觉图中的旧文案开发。** 旧图仅用于视觉质感参考；最新命名是“AI引擎”“产业布局”。
- **不要编造业务数据、案例、合作方、牌照、收益或市场成绩。** 缺失内容使用显式占位或隐藏模块。
- **先完善现有工程，不从零重写。** 除非经过技术审计确认当前结构不可维护。
- 每个阶段完成后运行类型检查、构建、自动测试与 Smoke Test，并输出：改动摘要、测试结果、未解决事项。
- 不将任何密码、Token、AccessKey、证件扫描件或私人联系方式写入仓库。

## 1. 产品概述

### 1.1 项目名称

**星橡 StarOak 集团官网**

### 1.2 公司主体

- 中文法定全称：**星橡（深圳）产业控股有限公司**
- 英文全称：**StarOak (Shenzhen) Industrial Holdings Co., Ltd.**
- 对外品牌：**StarOak 星橡控股**
- 域名：**www.staroakx.com**
- 初期部署：**香港区域**
- 后期部署：视业务与备案情况迁移或扩展至中国大陆

### 1.3 品牌定位

> **星橡是一家 AI 驱动的智能产业控股集团。**

星橡以 AI 引擎为集团级核心能力，通过技术研发、平台共建、参股联合、产业服务与生态协同，让 AI 进入真实产业场景，持续形成可复用、可治理、可扩展的长期价值。

### 1.4 核心主张

- 中文主标题：**用 AI 重构产业智能**
- 品牌主张：**智能为根，向星未来**
- 英文标语：**Rooted Intelligence. Starward Future.**

### 1.5 官网定位

官网不是普通企业介绍页，而是：

1. **集团战略信任中枢**：面向产业、资本、政府园区、合作伙伴、媒体与人才建立可信认知；
2. **品牌内容中枢**：统一承载集团定义、AI 引擎、产业布局、星橡智库和合规口径；
3. **生态合作中枢**：承接技术合作、产业服务、平台共建、参股联合与研究订阅；
4. **长期数字资产底座**：为英文站、产业详情、智库专题、项目公司页面和投资人专区预留扩展能力。

## 2. 已确认的核心战略架构

### 2.1 一核多翼，一院一网

- **一核：AI 引擎**  
  模型与智能体、数据智能、流程自动化、增长智能、量化研究辅助、可信 AI 治理。
- **多翼：产业与服务平台**  
  当前重点展示四个抓手；未来方向按成熟度逐步公开。
- **一院：星橡智库**  
  由原“星橡研究院 + 星橡洞察”合并而成，统一承载研究、方法论、观点、报告和管理层表达。
- **一网：星橡产业生态网**  
  连接技术、产业、资本、政府园区、客户、项目方与服务伙伴。

### 2.2 当前重点业务抓手

| 抓手 | 官网角色 | 已确认关系/口径 |
|---|---|---|
| **沐洋智联｜AI 全球增长服务平台** | 企业全球增长与产业服务抓手 | 星橡参股联合打造；承载市场洞察、内容生成、渠道触达、线索管理、AI 获客和增长运营 |
| **星藤智能科技 × 爬山虎｜AI 智能获客** | AI 获客主要内容 | 聚焦全域获客、AI 客服、会话洞察、意向客户识别、私域转化和运营提效 |
| **企业 AI 技术效能增长平台** | 企业内部 AI 效能方案 | 将 AI 从个人工具升级为组织级生产力系统，覆盖工作台、模型治理、研发 Agent、效能看板、资产沉淀与安全治理 |
| **APEX AI 量化交易平台** | AI 量化交易系统能力 | 以 AI 信号工厂、多市场执行中台、统一风控与审计为主要表达；必须使用风险提示 |

### 2.3 战略预留方向

仅作为长期空间低调展示，不做成熟业务承诺：

- 跨境智能服务；
- 文化科技；
- 数字员工；
- 低空智能；
- 产业专题与项目公司矩阵。

## 3. 产品目标与成功标准

### 3.1 业务目标

1. 在 30 秒内让访客理解“星橡是谁、AI 引擎是什么、当前有哪些真实抓手”；
2. 建立国际控股集团级的专业、可信、克制形象；
3. 让产业合作、技术合作、平台共建和研究订阅形成可追踪线索；
4. 为后续 CMS 内容运营、SEO、英文站和产业详情页建立稳定架构；
5. 以合规方式展示 AI 量化交易、AI 获客与全球增长服务，不做收益或效果承诺。

### 3.2 第一版可量化验收目标

这些是项目验收目标，而非行业事实承诺：

- 7 个一级页面、2 个法律页面和 1 个兼容跳转全部可用；
- PC、平板、手机和微信内置浏览器核心路径可完成；
- 三类表单支持校验、授权、提交、入库/预览模式和通知；
- Lighthouse 目标：桌面端 90+、移动端 85+；
- Core Web Vitals 目标：LCP < 2.5s、CLS < 0.1、INP < 200ms；
- S0/S1 缺陷在公开上线前清零；
- 关键页面具备独立 Metadata、Canonical、OG、Sitemap 与结构化数据。

## 5. 范围与非范围

### 5.1 第一版范围（P0）

- 7 个一级页面；
- 隐私政策、免责声明；
- `/ai-core` → `/ai-engine` 永久跳转；
- 响应式 Header、移动端菜单、Footer；
- 深穹星轨视觉系统与轻量动效；
- 三类 Lead 表单；
- Strapi 内容模型与静态内容 fallback；
- SEO、OG、Sitemap、Robots、Canonical、基础 JSON-LD；
- Docker/Nginx 香港部署方案；
- QA、备份、回滚和上线运行手册。

### 5.2 第一版明确不做

- 会员登录；
- 投资人后台；
- 实时交易或资产管理功能；
- 复杂数据看板；
- 完整英文站；
- 大型 WebGL/3D 或自动播放背景视频；
- 未确认的案例、数据、合作伙伴 Logo 墙；
- 未经合规审核的量化收益、获客转化和海外增长结果展示。

## 9. 视觉设计系统

### 9.1 视觉目标

**国际控股集团低奢感 / 深穹星轨**：大气、深邃、克制、秩序、资本感、长期主义。质量基准是国际顶级投行、咨询公司和控股集团官网，但不得照搬任何品牌。

### 9.2 核心图形语法

- StarOak Orbit Mark 的交织轨道；
- 钛银金属质感；
- 微光金战略星点；
- 深色空间、负空间和精密细线；
- 低速、低干扰的轨道运动；
- 数据与生态关系图使用简洁 SVG，而非炫技 3D。

### 9.3 颜色 Token

| Token | 值 | 用途 |
|---|---|---|
| Obsidian | `#050914` | 主背景 |
| Starfield Navy | `#0B1220` | 次背景、卡片层 |
| Panel | `rgba(13,21,35,.78)` | 玻璃卡片 |
| Titanium Silver | `#D9DCE1` | 主文字、Logo 辅助 |
| Soft Silver | `#A8ADB7` | 次文字 |
| Cloud White | `#F3EFE7` | 高亮文字 |
| Micro Gold | `#D5B36E` | CTA、星点、关键线条 |
| Deep Gold | `#9F8450` | 次级高亮 |

金色占比建议不超过 5%；不使用亮蓝作为集团官网主 CTA 或核心视觉色。

### 9.4 排版

- 中文正文：系统无衬线字体栈，优先 `PingFang SC / Microsoft YaHei / Noto Sans SC`；
- 英文正文：`Inter / system-ui`；
- 大标题：可使用具备高端感的衬线栈 `Noto Serif SC / Songti SC / Georgia`，但必须测试中文显示；
- 不依赖 Google Fonts 等大陆访问不稳定的第三方字体 CDN；
- 标题行距 1.04–1.15，正文行距 1.6–1.8。

### 9.5 禁止方向

- 亮蓝 SaaS 风；
- 机器人脸、芯片特写、握手、会议室等素材模板；
- 赛博朋克霓虹；
- 世界地图作为首屏主锚点；
- 重型 WebGL、粒子瀑布或自动播放大视频；
- 大面积金色、过度玻璃拟态或廉价高光。

视觉参考见 `docs/visual-references/`。其中旧图中的“AI智能底座”“产业矩阵”等文字已失效，只参考布局、质感与光影。

## 12. 响应式与交互要求

### 12.1 断点

- Wide desktop：≥1440px；
- Desktop：1024–1439px；
- Tablet：768–1023px；
- Mobile：≤767px；
- 验收重点：390×844、430×932、768×1024、1440×900。

### 12.2 移动端

- 必须实现真实可用的汉堡菜单，不得只隐藏桌面导航；
- Hero 单列，主 CTA 优先；
- 架构图和飞轮改为纵向叙事；
- 卡片单列或横向滑动需保证可访问；
- 表单字段单列；
- 不隐藏合规提示。

### 12.3 动效

- 轨道慢速旋转或漂移：单周期建议 12–24 秒；
- hover/focus：160–240ms；
- 页面入场：300–600ms，轻微位移与透明度；
- `prefers-reduced-motion: reduce` 时停用非必要动画；
- 不影响 LCP，不阻塞交互。

## 13. CMS 内容模型

使用 **Strapi 自托管 + PostgreSQL**，内容读取采用只读 Token，Lead 写入经 Next.js 服务端 API。

### 13.1 核心模型

1. `Page`：固定页面；
2. `Industry`：四个抓手和未来产业；
3. `Capability`：AI 引擎能力；
4. `Article`：星橡智库文章；
5. `Report`：报告/白皮书；
6. `Lead`：三类表单线索。

详细字段见 `docs/CMS_API_DEPLOYMENT_V3.4.md` 和 `specs/content-seed.json`。

### 13.2 发布流程

草稿 → 品牌审核 → 必要时合规审核 → 预览 → 发布 → 归档。

量化、获客、全球增长、个人资料处理内容必须进入合规审核。

### 13.3 Fallback

CMS 未配置或不可用时，前台必须继续使用 `lib/site-data.ts` 静态内容，不得导致集团首页不可访问。

## 16. 隐私、安全与合规

### 16.1 隐私

- 表单必须展示收集目的和隐私政策链接；
- 明确个人资料类型、用途、保存、访问/更正/删除方式；
- Lead 查看/导出独立授权；
- 不把线索数据发送到未经批准的第三方；
- 公开上线前由 CTO/法务复核香港部署阶段政策；迁移大陆时适配大陆个人信息保护要求。

### 16.2 安全

- HTTPS；
- 安全响应头、CSP（分阶段上线）、Referrer Policy、X-Content-Type-Options；
- API 速率限制；
- 后台强密码/2FA/IP 控制；
- PostgreSQL 每日备份；
- 对象存储版本控制；
- 日志不得记录完整敏感内容；
- 密钥只放环境变量或 Secret Manager。

### 16.3 文案禁区

禁止使用：保证收益、稳赚不赔、零风险、自动赚钱、保证获客、保证转化、全球第一、绝对领先、无需授权获取数据、已全面落地（对预留业务）。
